
## نظرة عامة

ويب أبليكيشن لإدارة المشاريع التجارية (مناقصات وأوامر مباشرة) بواجهة عربية RTL، مبني على **Lovable Cloud** (PostgreSQL + Auth + Storage مجاني)، استضافة Lovable مجانية مدى الحياة، مع تنبيهات إيميل (Resend عبر Lovable Email) وتيليجرام (Bot API). مراحل المشروع **قابلة للتخصيص بالكامل** من المدير.

---

## الأدوار والصلاحيات

- **Admin (مدير)**: يرى كل المشاريع، يدير الموظفين، ينشئ مشاريع وقوالب مراحل، يسند المهام، يرى كل التحديثات والملفات والمالية.
- **Employee (موظف)**: يرى فقط المهام المسندة إليه + التعليقات الخاصة بها، يحدّث حالة المهمة، يرفع ملفات، يضغط "إبلاغ عن عائق".

الأدوار مخزّنة في جدول `user_roles` منفصل (مش على profiles) لمنع رفع الصلاحيات.

---

## إدارة الموظفين (CRUD كامل للمدير)

- **إضافة موظف جديد**: المدير يُدخل (الاسم، الإيميل، رقم الموبايل، الدور، كلمة سر مبدئية) → يُنشأ حساب في Auth + سجل في `profiles` + دور في `user_roles` تلقائياً، ويصل للموظف إيميل ترحيب ببياناته.
- **تعديل بيانات موظف**: الاسم، الإيميل، الموبايل، الدور، Telegram chat ID، تفعيل/تعطيل الحساب.
- **إعادة تعيين كلمة السر**: زر يُرسل رابط reset للموظف.
- **حذف موظف**: حذف ناعم (Soft delete = تعطيل الحساب) بحيث تبقى سجلاته في المشاريع التاريخية سليمة، مع خيار **حذف نهائي** يمسحه من Auth ويُعيد إسناد مهامه المفتوحة لمدير قبل الحذف.
- **عرض الموظفين**: قائمة مع بحث وفلاتر (الدور، الحالة)، وعدد المهام النشطة لكل موظف.

كل ده محمي بـ RLS: فقط من له دور `admin` يقدر يستعمل عمليات الإدارة دي، عبر Server Functions تستخدم `supabaseAdmin` (service role) لإنشاء/حذف المستخدمين في Auth.

## الشاشات الرئيسية

### للمديرين
1. **Dashboard تنفيذي**: عدد المشاريع، مؤشر صحة كل مشروع (أخضر/أصفر/أحمر)، المهام المتأخرة، التنبيهات النشطة، عوائق مفتوحة.
2. **قائمة المشاريع** + فلاتر (الحالة، النوع، الموظف المسؤول، تاريخ).
3. **صفحة المشروع الكاملة**: بيانات، مخطط مراحل (Stepper/Gantt مبسّط)، Timeline، الملفات، تبويب مالي، تبويب مهام.
4. **إدارة الموظفين** (المذكورة أعلاه).
5. **قوالب المراحل (Workflow Templates)**: المدير يبني قوالب جاهزة، يضيف/يحذف/يرتّب الخطوات، يحدد لكل خطوة: الاسم، النوع (مرحلة تقدم / معلوماتية فقط)، المدة المتوقعة، هل تتطلب مرفقات/حقول مالية.
6. **التقارير المالية** لكل مشروع: تكلفة الشراء، التحصيل، صافي الربح.

### للموظفين
1. **My Tasks**: مفلترة بالحالة (جديدة، قيد التنفيذ، متأخرة، مكتملة).
2. **صفحة المهمة**: تفاصيل، Deadline، تحديث الحالة، رفع ملفات، تعليق، زر **"إبلاغ عن عائق"**.
3. **الإعدادات الشخصية**: ربط حساب Telegram، تعديل الموبايل، تغيير كلمة السر.

---

## مراحل المشروع (Dynamic Workflow Engine)

- عند إنشاء مشروع: المدير يختار قالب → النظام ينسخ خطواته للمشروع.
- المدير يقدر يعدّل خطوات هذا المشروع تحديداً (إضافة/حذف/إعادة ترتيب) بعد الإنشاء.
- كل خطوة لها: مسؤول، Deadline، حالة (انتظار/قيد العمل/مكتملة/معطّلة)، مرفقات، تعليقات، حقول مالية اختيارية.
- الخطوات المعلوماتية لا تؤثر على نسبة التقدم.

Seed قوالب جاهزة: قالب مناقصة (كراسة شروط → تأمين → عرض سعر → ترسية → أمر توريد → شراء → تسليم → محضر تسليم → أمر دفع → تحصيل) وقالب أمر مباشر (من عرض السعر).

---

## نظام التنبيهات التصاعدي

pg_cron داخل Lovable Cloud يفحص المهام كل دقيقة ويرسل عبر إيميل + تيليجرام:
- **عند الإسناد**: تنبيه فوري للموظف.
- **قبل الموعد بـ24 ساعة**: تنبيه ودّي.
- **عند الموعد بدون تحديث**: تنبيه عاجل للموظف.
- **بعد الموعد بـ24 ساعة**: تنبيه للمدير.
- **عند "إبلاغ عن عائق"**: العداد يتوقف + تنبيه فوري للمدير.

تيليجرام مجاني بالكامل (Telegram Bot API). كل موظف يكتب `/start` للبوت مرة واحدة لربط حسابه.

---

## المالية (مبسّطة في v1)

- كل مشروع: تكلفة شراء + قيمة تحصيل + حساب ربح تلقائي.
- تنبيه للمدير عند رفع محضر التسليم بضرورة متابعة الفاتورة.
- تقرير مالي إجمالي بسيط في Dashboard.

قابل للتوسعة لاحقاً (ضرائب، عملات متعددة، تقارير مفصّلة).

## هيكل قاعدة البيانات

| الجدول | الوصف |
|---|---|
| `profiles` | الاسم، الموبايل، telegram_chat_id، is_active |
| `user_roles` | الأدوار (admin / employee) — جدول منفصل |
| `workflow_templates` + `workflow_template_stages` | قوالب المراحل |
| `projects` | المشاريع + البيانات المالية |
| `project_stages` | خطوات كل مشروع (منسوخة من القالب، قابلة للتعديل) |
| `tasks` | المهام (مرحلة + موظف + Deadline + حالة) |
| `task_attachments` | ملفات في Lovable Storage |
| `task_comments` | تعليقات/تحديثات |
| `project_logs` | Timeline |
| `blockers` | العوائق |
| `notifications_queue` | طابور التنبيهات |

RLS مفعّل على كل الجداول مع `has_role()` security-definer function لمنع recursion.

## الحزمة التقنية

- **Frontend + Backend**: TanStack Start (SSR + Server Functions).
- **DB / Auth / Storage**: Lovable Cloud.
- **إيميل**: Lovable Email (Resend).
- **تيليجرام**: Telegram Bot API عبر Lovable connector.
- **جدولة**: pg_cron يستدعي route عام بـ HMAC signature.
- **استضافة**: Lovable مجانية مدى الحياة (`*.lovable.app` + SSL).
- **اللغة**: عربي RTL كامل.

## خطة التنفيذ

1. **الأساس**: Lovable Cloud + Auth + RTL + جداول + RLS + أول مدير + شاشة Login.
2. **إدارة الموظفين**: CRUD كامل (إضافة، تعديل، تعطيل، حذف، إعادة تعيين كلمة سر).

4. **المشاريع والمراحل**: إنشاء، تعديل، Stepper، صفحة كاملة للمدير + رؤية محدودة للموظف.
5. **المهام والمرفقات**: إسناد، رفع ملفات، تعليقات، Timeline، زر العائق.
6. **التنبيهات**: ربط Telegram + Email + pg_cron + المنطق التصاعدي.
7. **Dashboard والمالية**: مؤشر صحة، إحصائيات، تقرير ربح.
